# FSN-0021 UI Tweaks Implementation Plan

## Overview

Polish the public-facing surfaces of the TAURON demo: add hero/login background imagery, put the brand logo where a logo belongs, stop leaking "demo data" to end users, align avatar color with the brand, enrich landing-page content to reflect what the app actually does, and let chat widgets breathe at full column width.

## Current State Analysis

All changes are concentrated in `apps/main/src/`.

- **Landing** (`app/page.tsx`) — plain `bg-background` hero with no imagery, 3 generic feature cards, and a footer line `<span>Wersja demo — dane przykładowe</span>` (line 93) visible to end users.
- **Navbar** (`components/Navbar.tsx`) — renders `{BRAND.name}` as text only; uses hardcoded `border-gray-200` instead of the theme token.
- **Login** (`app/(auth)/login/page.tsx`) — bare white page with an inline server-action form. A nicer `<LoginForm />` component exists at `components/auth/login-form.tsx` wired to `loginAction` + `useActionState`, but it is orphaned.
- **Logo asset** (`branding/logo.svg`) — white-only paths (`fill="white"`); legible only on a colored backdrop.
- **Public assets** — `apps/main/public/` does not exist; Next.js `output: "standalone"` is configured (`next.config.ts:4`).
- **Railway build** (`apps/main/railway.toml:3`) — copies `.next/static` into `.next/standalone/apps/main/.next/static` but does not copy `public/`. Any new public asset would 404 in production.
- **Topbar avatar** (`components/layout/topbar.tsx:5-12, 36-41`) — `avatarColor(seed)` hashes `user.id` into a random hue via `oklch(0.55 0.18 ${hue})`; unrelated to the brand.
- **Chat** (`app/agent/chat.client.tsx`) — the per-turn wrapper (`max-w-2xl` column) is fine, but the widget group at line 133 is clamped to `max-w-[80%]`, so widgets render narrower than the text they illustrate.

## Desired End State

A visitor hitting `/` sees a branded hero with imagery, a logo top-left, an accurate feature list, and no "demo data" disclaimer. Visiting `/login` shows the same branded background, the logo, and the nicer form component. Logged-in users see their avatar in Tauron magenta. Inside the chat, tariff/consumption/contract widgets span the full chat column. Railway builds continue to deploy cleanly.

### Key Discoveries

- `BRAND` config (`branding/config.ts`) already centralizes name/tagline/theme — no new config needed.
- `LoginForm` (`components/auth/login-form.tsx:11-55`) is a drop-in replacement for the inline form; the matching server action is `loginAction` (`lib/actions/auth.action.ts:8-31`) and returns `LoginState` (`{ error?: string }`). The page-level `searchParams.error` branch becomes dead code once we switch.
- `TariffComparatorWidget` (`app/agent/widgets/tariff-comparator.client.tsx:61`) already uses `grid w-full grid-cols-1 md:grid-cols-3` internally — the 80% clamp at `chat.client.tsx:133` is what makes it look squashed.
- `logo.svg` viewBox is `0 0 84 60` with white fills — we'll use it inside a `bg-primary` rounded container (no asset edit).
- `next.config.ts` uses `output: "standalone"`. Next.js does **not** auto-copy `public/` into that output — the Railway build command must do it.

## What We're NOT Doing

- No new variants of `logo.svg` (colored/dark/mono) — the primary pill handles every context.
- No sourcing/committing of real `hero.jpg` / `login-bg.jpg` binaries — the user drops those in.
- No Tailwind theme changes beyond fixing `border-gray-200` → `border-border` in Navbar.
- No changes to widget component internals; only the chat wrapper that hosts them.
- No rework of the authenticated app shell or dashboard layout.
- No change to SEO/OpenGraph image generation.

## Implementation Approach

Four sequenced phases, smallest-blast-radius first. Phase 1 introduces the shared primitives (`<BrandLogo />`, `public/` dir, Railway fix) so Phases 2–3 can reuse them without churn. Phase 4 is the chat-side cosmetic fix, independent of the rest.

## Phase 1: Foundations — BrandLogo, public assets, Railway fix

### Overview

Create the shared logo pill, scaffold the `public/` directory so bg images can be referenced by path, and fix the Railway build so those images survive into the deployed container.

### Changes Required

#### 1. New reusable `BrandLogo` component

**File**: `apps/main/src/components/brand-logo.tsx` (new)
**Changes**: White `logo.svg` rendered inside a `bg-primary` rounded pill. Accepts `className` and a `size` prop to scale. Import the SVG as a React component via Next/Turbopack's static asset handling (`import Logo from "@/branding/logo.svg"` with Next 16's built-in SVG loader — falls back to `<Image />` + `import logoUrl from "..."` if SVGR isn't configured; verify during implementation and prefer the approach that requires no new dependency).

```tsx
import Image from "next/image";
import logoUrl from "@/branding/logo.svg";
import { cn } from "@/lib/utils";

export function BrandLogo({
  className,
  size = "md",
}: {
  className?: string;
  size?: "sm" | "md" | "lg";
}) {
  const dims = size === "lg" ? 48 : size === "sm" ? 24 : 32;
  const pad = size === "lg" ? "px-4 py-2" : size === "sm" ? "px-2 py-1" : "px-3 py-1.5";
  return (
    <span
      className={cn(
        "inline-flex items-center justify-center rounded-md bg-primary",
        pad,
        className,
      )}
    >
      <Image
        src={logoUrl}
        alt="TAURON"
        height={dims}
        width={Math.round(dims * 1.4)}
        priority
      />
    </span>
  );
}
```

If direct `import` of `.svg` conflicts with the existing `turbopack.rules` for `.md`, fall back to placing a copy of `logo.svg` at `apps/main/public/logo.svg` and using `<Image src="/logo.svg" ... />`.

#### 2. Scaffold `apps/main/public/`

**File**: `apps/main/public/.gitkeep` (new)
**Changes**: Empty file to keep the dir in git.

**File**: `apps/main/public/README.md` (new)
**Changes**: One-paragraph note instructing that `hero.jpg` (≥1920×1080, landscape, dark-friendly subject) and `login-bg.jpg` (≥1600×1200, portrait-or-landscape OK, dark-friendly) must be provided before shipping. Include a TODO marker so `grep -r TODO apps/main/public` surfaces the dependency.

#### 3. Railway build — copy `public/` into standalone output

**File**: `apps/main/railway.toml`
**Changes**: Extend `buildCommand` with a `public/` copy step.

```toml

```

The `cp` will fail the build if `public/` is absent — the `.gitkeep` in step 2 guarantees presence.

## Phase 3: Login page

### Overview

Rebuild `/login` to use the existing `<LoginForm />` component, on top of a bg image, with the brand logo visible.

### Changes Required

#### 1. Replace inline form with `<LoginForm />`

**File**: `apps/main/src/app/(auth)/login/page.tsx`
**Changes**: Full rewrite. Keep the existing `auth()` redirect-when-signed-in guard; drop the inline `<form action={...}>` and the `searchParams.error` branch (errors now surface inside `LoginForm` via `useActionState`).

```tsx
import { redirect } from "next/navigation";
import { auth } from "@/auth";
import { BRAND } from "@/branding/config";
import { BrandLogo } from "@/components/brand-logo";
import { LoginForm } from "@/components/auth/login-form";

export default async function LoginPage() {
  const session = await auth();
  if (session?.user) redirect("/app/dashboard");

  return (
    <div className="relative isolate flex min-h-screen items-center justify-center px-4 py-10 bg-primary/10">
      <div
        aria-hidden
        className="absolute inset-0 -z-10 bg-[url('/login-bg.jpg')] bg-cover bg-center"
      />
      <div
        aria-hidden
        className="absolute inset-0 -z-10 bg-gradient-to-br from-black/55 to-black/25"
      />
      <div className="w-full max-w-sm rounded-xl bg-card/95 p-8 shadow-xl backdrop-blur-sm card-shadow">
        <div className="mb-6 flex flex-col items-center gap-3 text-center">
          <BrandLogo size="lg" />
          <h1 className="text-xl font-semibold">{BRAND.fullName}</h1>
          <p className="text-sm text-muted-foreground">Zaloguj się, aby kontynuować</p>
        </div>
        <LoginForm />
      </div>
    </div>
  );
}
```

No change to `LoginForm` itself or to `loginAction`.

## Performance Considerations

- Hero and login bg images are the largest new payloads. Ask for JPEGs (not PNG) and recommend ≤300 KB each; add `loading="eager"` via CSS background (no `<img>` needed for decorative bg).
- `BrandLogo` uses `priority` on the Next `<Image />` for above-fold rendering — avoids LCP regression on `/` and `/login`.
- No runtime changes, no new dependencies.

## Migration Notes

None. All changes are additive or in-place UI edits; no DB migrations, no env var changes, no auth flow changes.
