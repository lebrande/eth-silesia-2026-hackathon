# Branding Customization Implementation Plan

> Ticket: [`thoughts/tickets/fsn_0003-branding.md`](../tickets/fsn_0003-branding.md)
> Target tenant: **Tauron Polska Energia** (reference: [tauron.pl/dla-domu](https://www.tauron.pl/dla-domu), [raport.tauron.pl](https://raport.tauron.pl))

## Overview

Centralize all brand-dependent assets (logo, favicons, product name, tagline, SEO metadata, primary color, font) into a single `apps/main/src/branding/` folder driven by one typed config file. Replace every hard-coded "Eth Silesia" / "Backoffice AI" string and the default purple theme with values from that config. First applied preset is Tauron (magenta `#e2007a`, Titillium Web, Tauron favicons/logo).

A developer changing brand should only need to edit one folder.

## Current State Analysis

- Brand strings are hard-coded in 9 places — layout metadata title, homepage hero, Navbar, Sidebar (two labels), login page heading, login-form default email, auth.ts `ADMIN_EMAIL`, `verify-phone.node.ts` SMS prefix, plus `apps/main/README.md`.
- Favicon: single `apps/main/src/app/favicon.ico` (Next.js app-router convention). No `apple-icon`, `icon`, `opengraph-image`, manifest, or robots.txt.
- No `apps/main/public/` directory exists.
- Metadata: only `title: "Eth Silesia"` in `app/layout.tsx:4-6`. No description, no openGraph, no themeColor.
- Font: Inter loaded via `<link>` in `app/layout.tsx:15-25`; `--font-sans` set to `"Inter"` in `globals.css:22`.
- Primary color: `--color-primary: oklch(0.55 0.22 280)` (purple) in `globals.css:13`. The chat widget hard-codes `bg-blue-600` / `hover:bg-blue-700` instead of using the Tailwind primary token.
- No tenant-specific config file exists anywhere.

### Key Discoveries

- Next.js 16 app-router supports special files `icon.{ico,png}`, `apple-icon.png`, `opengraph-image.png` directly in `app/` — they become `/icon`, `/apple-icon`, `/opengraph-image` automatically. No manual `<link rel>` needed; the framework injects tags based on the `metadata` export.
- `metadata` can also be an async function/object using [`Metadata`](https://nextjs.org/docs/app/api-reference/file-conventions/metadata) — we import config values directly.
- Tailwind v4 theme tokens (`@theme { --color-primary: ... }` in `globals.css`) are what drive `bg-primary`, `text-primary`, etc. Changing that one line re-themes the whole app for components that use the token.
- `ChatWidget.client.tsx:82,112,125` hard-codes blue classes — these must switch to `bg-primary` tokens, otherwise the chat button stays blue on a magenta brand.
- Tauron primary color `#e2007a` (sourced from `meta[theme-color]` and `--agb-c-act`). Font `Titillium Web` (300/400/600/700 latin-ext) via Google Fonts. Copy: "TAURON Polska Energia", description "TAURON Polska Energia to dostawca prądu dla domów i firm".
- `app/globals.css:1` has `@import "tailwindcss"` — any new CSS variables live inside the existing `@theme` block.

## Desired End State

```
apps/main/src/branding/
  config.ts          ← typed config (name, titles, copy, colors, font, SEO)
  logo.svg           ← brand logo (vector, used in Navbar/Sidebar/login)
  icon.png           ← 512×512, feeds favicon + all icon sizes
  apple-icon.png     ← 180×180 Apple touch icon
  opengraph-image.png ← 1200×630 OG share image
  README.md          ← 5-line guide: "edit these files to rebrand"
```

The old `apps/main/src/app/favicon.ico` is deleted. New `app/icon.tsx`, `app/apple-icon.tsx`, `app/opengraph-image.tsx` stream bytes from `src/branding/*` (keeps the "one folder" invariant).

Verification:
- `pnpm -F main dev` boots, `/` shows "TAURON Polska Energia" in hero, magenta navbar/buttons, Titillium Web body text.
- Browser tab shows Tauron favicon and title "TAURON Polska Energia".
- `view-source:localhost:3000` includes `<meta name="description">`, `<meta property="og:image">`, `<meta name="theme-color" content="#e2007a">`.
- Changing `config.ts` `name: "Foo"` and replacing `logo.svg` re-brands the app with no other code edits.
- `pnpm -F main typecheck` passes; `pnpm -F main lint` clean; `pnpm -F main build` succeeds.

## What We're NOT Doing

- No multi-tenant runtime switching (no DB-backed branding, no per-request brand lookup). `config.ts` is a compile-time constant.
- No admin UI to edit branding. Devs edit files directly.
- No dark mode or secondary-color tokens beyond what already exists in `globals.css`.
- No change to LangGraph agent prompts (`agent-shared.prompt.md` already says "Tauron Polska Energia") and no customer-facing prompt re-brand.
- No `manifest.json` / PWA support, no `robots.txt`, no sitemap.
- No logo component animations, no logomark/wordmark split — one SVG file covers all usage sites.
- No backwards-compatibility shim for the removed `favicon.ico`.
- Root `package.json` `"name": "eth-silesia"` and monorepo identifiers stay as-is (they're not user-facing).
- Backoffice agent system prompt reference in `lib/agent/prompt.ts:47` stays hard-coded — it's a separate tenant concern (customer service persona).

## Implementation Approach

One config object (`BRAND`) re-exported from `@/branding/config` drives every touchpoint. Assets live next to it. The `app/` tree owns only thin re-exports and consumes `BRAND`. Tailwind theme tokens in `globals.css` and the Google Fonts `<link>` in `layout.tsx` are updated once to match Tauron's palette/typography.

## Phase 1: Branding folder + typed config

### Overview

Create the central folder with a typed config object and a README. No wiring yet.

### Changes Required

#### 1. New file: `apps/main/src/branding/config.ts`

```typescript
// Single source of truth for all brand-dependent values.
// To rebrand: edit this file + replace the asset files in this folder.

export const BRAND = {
  // Product identity
  name: "TAURON",
  fullName: "TAURON Polska Energia",
  tagline: "Dostawca prądu dla domów i firm",

  // SEO / HTML metadata
  seo: {
    title: "TAURON Polska Energia",
    titleTemplate: "%s | TAURON",
    description:
      "TAURON Polska Energia to dostawca prądu dla domów i firm. Asystent obsługi klienta.",
    locale: "pl_PL",
    siteUrl: "https://tauron.pl",
  },

  // Brand hub labels (sidebar / navbar pair)
  backoffice: {
    productLabel: "Backoffice AI",
    tenantLabel: "TAURON",
  },

  // Auth demo / seed values
  auth: {
    adminEmail: "admin@tauron.pl",
    defaultLoginEmail: "admin@tauron.pl",
  },

  // SMS prefix for verification messages
  sms: {
    senderTag: "TAURON",
  },

  // Theme
  theme: {
    // Tauron magenta. Mirrored in globals.css --color-primary and used as
    // <meta name="theme-color">.
    primary: "#e2007a",
  },

  // Typography — Google Fonts family name + weights.
  font: {
    family: "Titillium Web",
    weights: [300, 400, 600, 700],
    subsets: ["latin", "latin-ext"],
  },
} as const;

export type Brand = typeof BRAND;
```

#### 2. New file: `apps/main/src/branding/README.md`

```markdown
# Branding

Single source of truth for the app's brand identity.

To rebrand the app:

1. Edit `config.ts` — copy, colors, font.

3. Update `--color-primary` in `app/globals.css` to match `theme.primary`.
```

## Phase 2: Wire metadata, favicons, and OG image

### Overview

Replace the Next.js favicon file with app-router `icon.tsx` / `apple-icon.tsx` / `opengraph-image.tsx` stubs that stream bytes from `src/branding/`. Update the root layout's `metadata` export to populate title, description, themeColor, openGraph, and twitter from `BRAND`.

### Changes Required

#### 1. Delete `apps/main/src/app/favicon.ico`

Next.js picks up the new `icon.*` file automatically once `favicon.ico` is gone.

#### 2. New file: `apps/main/src/app/icon.tsx`

```typescript
import { readFile } from "fs/promises";
import path from "path";

export const size = { width: 512, height: 512 };
export const contentType = "image/png";

export default async function Icon() {
  const file = await readFile(
    path.join(process.cwd(), "src/branding/icon.png"),
  );
  return new Response(file, {
    headers: { "Content-Type": contentType },
  });
}
```

#### 3. New file: `apps/main/src/app/apple-icon.tsx`

Same shape as `icon.tsx`, reads `src/branding/apple-icon.png`, `size = { width: 180, height: 180 }`.

#### 4. New file: `apps/main/src/app/opengraph-image.tsx`

Same shape, reads `src/branding/opengraph-image.png`, `size = { width: 1200, height: 630 }`.

#### 5. Rewrite `apps/main/src/app/layout.tsx`

```typescript
import type { Metadata, Viewport } from "next";
import { BRAND } from "@/branding/config";
import "./globals.css";

const fontQuery = [
  `family=${BRAND.font.family.replace(/ /g, "+")}`,
  `:wght@${BRAND.font.weights.join(";")}`,
  `&subset=${BRAND.font.subsets.join(",")}`,
  `&display=swap`,
].join("");

export const metadata: Metadata = {
  title: {
    default: BRAND.seo.title,
    template: BRAND.seo.titleTemplate,
  },
  description: BRAND.seo.description,
  applicationName: BRAND.name,
  openGraph: {
    title: BRAND.seo.title,
    description: BRAND.seo.description,
    siteName: BRAND.name,
    locale: BRAND.seo.locale,
    type: "website",
  },
  twitter: {
    card: "summary_large_image",
    title: BRAND.seo.title,
    description: BRAND.seo.description,
  },
};

export const viewport: Viewport = {
  themeColor: BRAND.theme.primary,
};

export default function RootLayout({
  children,
}: Readonly<{ children: React.ReactNode }>) {
  return (
    <html lang={BRAND.seo.locale.split("_")[0]}>
      <head>
        <link rel="preconnect" href="https://fonts.googleapis.com" />
        <link rel="preconnect" href="https://fonts.gstatic.com" crossOrigin="" />
        <link
          href={`https://fonts.googleapis.com/css2?${fontQuery}`}
          rel="stylesheet"
        />
      </head>
      <body className="font-sans">{children}</body>
    </html>
  );
}
```

## Testing Strategy

### Manual Testing Steps

1. `pnpm -F main dev` — visit `http://localhost:3000`, confirm hero/navbar.
2. Click "Zaloguj" → login page shows Tauron heading + prefilled email.
3. Log in as admin → `/app/dashboard` loads, sidebar shows Tauron labels.
4. Open the public chat widget on `/` → all buttons are magenta.
5. In devtools: Network tab → `/icon`, `/apple-icon`, `/opengraph-image` all return 200 + PNG.
6. View page source → confirm description/og/theme-color meta tags present.
7. Swap `BRAND.name` to `"Foo"` temporarily → confirm all UI touchpoints flip to "Foo". Revert.

### No automated tests

The project has no test runner configured beyond `tsc --noEmit` and `eslint`. Not adding one for this task — YAGNI.

## Performance Considerations

`icon.tsx` reads from disk on every request. Next.js caches responses per route in production by default (route handlers default to `force-static` at build time for files without dynamic segments). Acceptable for a hackathon; no tuning needed.

## Migration Notes

- Deleting `app/favicon.ico` will briefly invalidate cached browser favicons. Hard-refresh in testing.
- The existing `app/layout.tsx` edits the font `<link>` — users hitting the page during deploy may see Inter flash before Titillium Web loads. Negligible.
- No DB changes, no env changes, no Railway config changes.
